# アーキテクチャ概要

> 本ドキュメントは設計の **現時点の方針** を残すもの。実装の進捗に合わせて更新する。

## 目的

FX為替予測アプリ。主戦略判断は 1分足 / 5分足、秒単位監視は執行補助扱い。
初期は「予測・分析・可視化・検証」が優先、自動発注は後段で安全に実装。

## レイヤ構成 (モノレポ)

```
src/fx_ai_trading/
├── app/         # (将来) Web API / CLI エントリポイント
├── services/    # ドメインサービス: データ取得/保存/特徴量/予測/シグナル/執行
├── batch/       # 学習/集計など手動・スケジュール実行のタスク
├── dashboard/   # 可視化・管理画面
└── common/      # 設定・ログなど横断ユーティリティ
```

## Phase ロードマップ

1. データ取得
2. データ保存
3. 特徴量生成
4. 予測
5. バックテスト
6. 可視化ダッシュボード
7. 執行補助
8. 自動売買 (後段・安全重視)

各 Phase は最小実装 → 検証 → 改善のサイクルで進める。Phase 間は層の境界で分離し、
次 Phase への移行時に前 Phase を破壊しないことを最優先。

## データストア

- **一次DB (PostgreSQL)**: 生データ・中間データを広めに保存
- **二次DB (PostgreSQL)**: 集計・分析用のビュー/マート

一次/二次の分離により、壊れにくい生データを保ちつつ、分析側を気軽に作り直せる。

## テスト/ハーネス方針

- 本番API直結の前に、モック・テストハーネスで検証する
- データ取得 / 特徴量 / 予測 / シグナル / 発注 の各層は Interface 化して単体テスト可能に
- バックテスト用データ、疑似リアルタイムデータ、モックブローカーを差し込める構造に保つ

## 設定・秘密管理

- 設定値は `.env` or 設定ファイル経由 (`src/fx_ai_trading/common/config.py` で一元化)
- 秘密情報はコード直書き禁止。`.env` は gitignored、共有は `.env.example` 経由
